# Инструкция по первому развертыванию (Deployment Guide)

Этот документ описывает пошаговый процесс развертывания приложения ProcessMeter с нуля.

## Предварительные требования
*   **Node.js** (версия 18+ рекомендуется)
*   **PostgreSQL** (установленный и запущенный сервер базы данных)
*   Доступ к эталонной базе данных (откуда будут копироваться справочники процессов)

## Шаг 1: Настройка окружения
1.  Скопируйте файл `.env.example` в `.env` в корне проекта (если его нет).
    ```bash
    cp .env.example .env
    ```
2.  Откройте `.env` и настройте переменные подключения:
    *   `TARGET_DATABASE_URL`: Строка подключения к вашей *целевой* базе данных (куда приложение будет писать данные).
    *   `SOURCE_DATABASE_URL`: Строка подключения к *эталонной* базе данных (источник справочников).
    *   `JWT_SECRET`: Секретный ключ для токенов (придумайте сложную строку).
    *   `PORT`: Порт для запуска бэкенда (по умолчанию 3000).

## Шаг 2: Установка зависимостей
Установите зависимости для всего проекта (корневая папка, бэкенд и фронтенд).
Выполните команду в корне проекта:
```bash
npm install
# Далее перейдите в папки и установите зависимости там, если npm install в корне не настроен через workspaces (в данном проекте workspaces не используются, поэтому нужно устанавливать отдельно)
cd backend && npm install
cd ../frontend && npm install
cd ..
```
*Примечание: Если `package.json` в корне настроен с workspaces, достаточно `npm install` в корне. В текущей структуре лучше убедиться, что `node_modules` есть везде.*

## Шаг 3: Инициализация базы данных и миграция
Запустите скрипт миграции. Этот скрипт подключится к `SOURCE_DATABASE_URL`, заберет оттуда справочники (process_1...4, systems, executors) и зальет их в `TARGET_DATABASE_URL`. Также он создаст необходимые таблицы.

Из корня проекта:
```bash
node backend/scripts/migrate.js
```
*Или через npm из папки backend:* `npm run db:migrate`

**Что проверить:**
В консоли должно появиться подробное логирование (Detailed Logging):
*   ✅ "Синхронизация таблицы: process_4... Вставлено строк..." (не должно быть 0, если в источнике есть данные).
*   ✅ "Итоговая статистика: Операций 4 уровня (process_4): X".

## Шаг 4: Создание администратора
Создайте первого пользователя-администратора для входа в систему.

```bash
# Формат: node backend/scripts/create-user.js <username> <password> "<Full Name>" admin
node backend/scripts/create-user.js admin password123 "Администратор Системы" admin
```
Запомните логин (`admin`) и пароль (`password123`).

## Шаг 5: Запуск приложения

### Режим разработки (Backend + Frontend)
Для разработки удобно запустить бэкенд и фронтенд в разных терминалах.

Терминал 1 (Backend):
```bash
cd backend
npm run dev
# или node src/server.js
```

Терминал 2 (Frontend):
```bash
cd frontend
npm run dev
```
Фронтенд обычно доступен по адресу: `http://localhost:5173` (или как укажет Vite).

## Возможные проблемы и решения

### "Ошибка: не создались операции 4 уровня"
Если после входа в приложение вы не видите операций:
1.  Посмотрите логи выполнения `migrate.js`.
2.  Найдите строку `Синхронизация таблицы: process_4`.
    *   Если написано "В источнике пуста" — значит в `SOURCE_DATABASE_URL` нет данных. Проверьте правильность подключения к источнику.
    *   Если "Вставлено строк: 0" — значит данные не скопировались.
3.  Проверьте "Итоговая статистика" в конце лога миграции.

### "Ошибка: ответы пользователей не создались"
Таблица `user_answers` заполняется на основе:
1.  Прав доступа пользователя к `process_1` (`user_process_1_access`).
2.  Наличия операций в `process_4`.
3.  Функции `copy_operations_to_user_answers`.

**Решение:**
1.  Убедитесь, что миграция прошла успешно (см. выше).
2.  Убедитесь, что пользователь (которого вы создали или используете) имеет права доступа. В админке (если доступна) или через БД проверьте таблицу `user_process_1_access`.
3.  Если прав нет, администратор должен выдать их через веб-интерфейс. После выдачи прав ответы сгенерируются автоматически.

### Полный сброс
Если нужно начать с чистого листа (удалить все данные):
```bash
node backend/scripts/clear-db.js
```
⚠️ Это удалит ВСЕХ пользователей и ВСЕ ответы.
