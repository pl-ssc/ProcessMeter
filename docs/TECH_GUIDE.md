# Инструкция для технического специалиста

В данном документе описаны процедуры развертывания, настройки и обслуживания приложения ProcessMeter.

## 1. Системные требования

*   Docker и Docker Compose.
*   Node.js v24+ (для локальной разработки).
*   Порты 3001 (backend), 5432 (database), 5433 (ref-database) должны быть свободны.

## 2. Развертывание базы данных

Приложение использует две базы данных:
1.  `my-postgres` (порт 5432) — рабочая база приложения.
2.  `ref-postgres` (порт 5433) — эталонная база для синхронизации справочников.

Запуск контейнеров:
```bash
docker-compose up -d
```

## 3. Настройка окружения (`.env`)

Создайте файл `.env` в корне проекта на основе `.env.example`. Основные параметры:
*   `DATABASE_URL`: Строка подключения к рабочей базе (`postgresql://postgres:postgres@127.0.0.1:5432/mydb`).
*   `SOURCE_DATABASE_URL`: Строка подключения к эталонной базе (`postgresql://postgres:postgres@127.0.0.1:5433/refdb`).
*   `ADMIN_USERNAME`, `ADMIN_PASSWORD`: Учетные данные для первичного создания администратора.
*   `AUTO_MIGRATE=true`: Позволяет серверу автоматически создать схему при первом запуске.

## 4. Миграция данных из эталона

Для синхронизации справочников и подготовки анкет пользователей используйте скрипт миграции.

**Запуск миграции из корня проекта**:
```bash
# Нужно зайти в папку backend
cd backend
# Загрузить переменные и запустить скрипт
sh -c 'set -a; source ../.env; set +a; npm run db:migrate'
```

**Что делает скрипт**:
1.  Сохраняет текущие права доступа пользователей.
2.  Очищает таблицы справочников и ответов.
3.  Копирует свежие данные из эталонной базы (5433).
4.  Восстанавливает права доступа.
5.  Генерирует новые пустые записи в `user_answers` для активных пользователей.

## 5. Запуск приложения в режиме разработки

Из корня проекта:
```bash
npm run dev:all
```
Это запустит одновременно Backend (3001) и Frontend (Vite).

## 6. Ручные операции (SQL)

Если автоматическая миграция не сработала, вы можете применить схему вручную через Docker:

```bash
docker exec -i my-postgres psql -U postgres -d mydb < db/schema.sql
```

Создание администратора вручную:
Скрипт создания администратора запускается автоматически при старте сервера `server.js` (функция `ensureAdminUser`), основываясь на данных из `.env`.

## 7. Устранение неполадок

*   **Ошибка "Relation already exists"**: Возникает, если схема уже создана, а `AUTO_MIGRATE=true`. Можно игнорировать или отключить флаг.
*   **Зависание при миграции**: Проверьте, нет ли активных сессий к базе данных. В случае блокировок выполните `docker-compose restart`.
*   **Ошибки ES-модулей в Node.js 24**: Проект настроен как `"type": "module"`. При добавлении новых скриптов используйте явные расширения файлов `.js`.
