# Руководство по BI-аналитике (Metabase)

Данный документ описывает структуру аналитической системы, развернутой для проекта **ProcessMeter**. Система построена на базе Metabase и использует специально подготовленные SQL-представления (Views) для упрощения бизнес-анализа.

## 1. Источники данных (SQL Views)

Для дашбордов используются три «плоских» представления, которые объединяют данные из нормализованных таблиц (процессы, ответы, пользователи).

### [view_bi_labor_distribution]
**Цель**: Основной источник для анализа трудозатрат и проектирования ОЦО (SSC).
- **fte**: Автоматически рассчитываемый эквивалент полной занятости (`labor_hours / 165`).
- **labor_hours**: Трудозатраты в **часах**.
- **executor_type**: Категория исполнителя (Фронт, Мидл, Бэк-офис).
- **process_level_1-4**: Полная иерархия процесса для группировки.
- **system_name**: Используемая ИТ-система.

### [view_bi_respondent_stats]
**Цель**: Операционный контроль хода опроса и анализ скорости заполнения.
- **invitation_at**: Дата и время создания пользователя (момент приглашения).
- **first_answer_at**: Дата первого внесенного ответа в анкету.
- **completion_at**: Дата нажатия кнопки «Завершить».
- **elapsed_days_since_invitation**: Время от приглашения до завершения в **днях**.
- **active_filling_days**: Реальное время в работе в **днях** (от первого ответа до завершения).
- **total_labor_hours**: Суммарные трудозатраты пользователя в **часах**.
- **completion_percentage**: Процент заполнения анкеты.
- **is_survey_completed**: Флаг финальной отправки анкеты.

### [view_bi_process_efficiency]
**Цель**: Поиск неэффективностей.
- **avg_labor_hours**: Среднее время на операцию в **часах**.
- **notes_count**: Количество комментариев (сигналы о проблемах).

### [view_bi_survey_progress_total]
**Цель**: Создание прогресс-бара (Индикатора выполнения).
- **total_respondents**: Всего приглашенных пользователей (роль 'respondent').
- **completed_respondents**: Количество пользователей, завершивших опрос.

## 2. Структура Дашборда «Управление ОЦО»

### Виджет 1: Прогресс заполнения анкет (%)
- **Источник**: `view_bi_respondent_stats`
- **Тип**: Scalar (Число)
- **Настройка**: Среднее (`Average`) значение по колонке `completion_percentage`.

### Виджет 2: Распределение FTE по подразделениям
- **Источник**: `view_bi_labor_distribution`
- **Тип**: Pie Chart (Круговая диаграмма)
- **Настройка**: 
    - **Значение (Metric)**: Сумма (`Sum`) по колонке `fte`.
    - **Группировка (Group by)**: `executor_type`.

### Виджет 3: Топ-10 процессов по трудозатратам
- **Источник**: `view_bi_labor_distribution`
- **Тип**: Bar Chart (Гистограмма)
- **Настройка**:
    - **Ось X**: `process_level_1`.
    - **Ось Y**: Сумма (`Sum`) по колонке `fte`.
    - **Сортировка**: По убыванию значения Y.

## 3. Ручная корректировка (если графики не отображаются)

Если после автоматического создания карточка просит «выбрать колонки»:
1. Нажмите иконку карандаша (**Edit Dashboard**) на дашборде.
2. Нажмите **Edit Query** на проблемном виджете.
3. В разделе **Visualization** выберите нужный тип графика (Pie/Bar/Scalar).
4. Убедитесь, что выбраны правильные колонки для осей (указаны в разделе 2).
5. Нажмите **Save**, чтобы обновить вопрос.
