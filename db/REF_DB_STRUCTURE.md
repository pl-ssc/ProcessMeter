# Структура эталонной базы данных (Reference DB / MDM)

Эталонная база данных (`refdb`) выступает единым источником правды для микросервисов и приложений, работающих с процессами. В ней хранятся **только справочные данные** (мастер-данные), исключая любые транзакционные или пользовательские (операционные) таблицы.

Ниже приведено описание таблиц, полей и их типов.

---

## 1. Иерархия бизнес-процессов

### Таблица `process_1` (Бизнес-процессы L1)
Верхнеуровневые направления деятельности (например: HR, IT, Бухгалтерия).
*   `f1_index` (text, Primary Key) — Уникальный код процесса.
*   `f1_name` (text, NOT NULL) — Название процесса (обязательное поле).
*   `note` (text) — Дополнительное описание или пояснение.
*   `is_active` (boolean, NOT NULL) — Флаг активности процесса (если `false`, процесс считается архивным). Значение по умолчанию: `true`.
*   `sort` (integer) — Порядковый номер для сортировки (например, при отображении в UI).

### Таблица `process_2` (Бизнес-процессы L2)
Декомпозиция процессов L1.
*   `f2_index` (text, Primary Key) — Уникальный код процесса L2.
*   `f1_index` (text, Foreign Key, NOT NULL) — Строгая ссылка на родительский процесс из `process_1`.
*   `f2_name` (text, NOT NULL) — Название процесса (обязательное поле).
*   `sort` (integer) — Порядковый номер для сортировки.
*   `note` (text) — Дополнительное описание.
*   `is_active` (boolean, NOT NULL) — Флаг активности. Значение по умолчанию: `true`.

### Таблица `process_3` (Бизнес-процессы L3)
Декомпозиция процессов L2.
*   `f3_index` (text, Primary Key) — Уникальный код процесса L3.
*   `f2_index` (text, Foreign Key, NOT NULL) — Строгая ссылка на родительский процесс из `process_2`.
*   `f3_name` (text, NOT NULL) — Название процесса (обязательное поле).
*   `sort` (integer) — Порядковый номер для сортировки.
*   `note` (text) — Дополнительное описание.
*   `is_active` (boolean, NOT NULL) — Флаг активности. Значение по умолчанию: `true`.

### Таблица `process_4` (Операции L4)
Самый детальный уровень процессов — конечные операции. Именно этот уровень обычно замеряется или выполняется конечными исполнителями.
*   `f4_index` (text, Primary Key) — Уникальный код операции L4.
*   `f3_index` (text, Foreign Key, NOT NULL) — Строгая ссылка на родительский процесс из `process_3`.
*   `f4_name` (text, NOT NULL) — Подробное описание операции (обязательное поле).
*   `sort` (integer) — Порядковый номер для сортировки.
*   `note` (text) — Дополнительное описание.
*   `is_active` (boolean, NOT NULL) — Флаг активности. Значение по умолчанию: `true`.
*   `executor_id` (integer, Foreign Key) — Идентификатор исполнителя из таблицы `executors`. Может быть `NULL`, если исполнитель не задан жестко.

---

## 2. Справочники

### Таблица `executors` (Исполнители / Роли)
Типы подразделений или профили ролей, выполняющих процесс на уровне L4.
*   `id` (integer, Primary Key) — Уникальный числовой ID исполнителя.
*   `name` (text, NOT NULL) — Название исполнителя (например, "Мидл-офис", "Бухгалтер по зарплате"). Обязательное поле.
*   `note` (text) — Дополнительное текстовое описание зоны ответственности исполнителя.

### Таблица `systems` (Информационные системы)
Список целевых ИТ-систем и программного обеспечения, в которых выполняются операции (например, 1С, SAP ERP, MS Excel).
*   `system_id` (integer, Primary Key) — Автоинкрементный уникальный идентификатор (IDENTITY).
*   `system_name` (text, NOT NULL, UNIQUE) — Уникальное название конфигурации или системы.
*   `is_active` (boolean, NOT NULL) — Флаг актуальности системы. Значение по умолчанию: `true`.
