# Структура базы данных ProcessMeter

В проекте используется PostgreSQL. База данных разделена на справочники (эталонные данные) и операционные таблицы (ответы пользователей).

## Справочники (Эталонные данные)

Эти таблицы содержат иерархию процессов и сопутствующую информацию. Они синхронизируются из эталонной базы.

### Таблица `process_1` (Бизнес-процессы L1)
Верхнеуровневые направления деятельности.
*   `f1_index` (text, PK): Уникальный код процесса (например, `ACC`, `HR`).
*   `f1_name` (text, NOT NULL): Название процесса.
*   `note` (text): Дополнительное описание.
*   `is_active` (boolean, NOT NULL): Статус активности (по умолчанию `true`).
*   `sort` (integer): Порядковый номер для сортировки.

### Таблица `process_2` (Бизнес-процессы L2)
*   `f2_index` (text, PK): Уникальный код процесса.
*   `f1_index` (text, FK, NOT NULL): Ссылка на родительский процесс L1.
*   `f2_name` (text, NOT NULL): Название процесса.
*   `sort` (integer): Порядок сортировки.
*   `note` (text): Примечание.
*   `is_active` (boolean, NOT NULL): Статус активности (по умолчанию `true`).

### Таблица `process_3` (Бизнес-процессы L3)
*   `f3_index` (text, PK): Уникальный код процесса.
*   `f2_index` (text, FK, NOT NULL): Ссылка на родительский процесс L2.
*   `f3_name` (text, NOT NULL): Название процесса.
*   `sort` (integer): Порядок сортировки.
*   `note` (text): Примечание.
*   `is_active` (boolean, NOT NULL): Статус активности (по умолчанию `true`).

### Таблица `process_4` (Операции L4)
Самый детальный уровень процессов, который оценивают пользователи.
*   `f4_index` (text, PK): Уникальный код операции.
*   `f3_index` (text, FK, NOT NULL): Ссылка на родительский процесс L3.
*   `f4_name` (text, NOT NULL): Описание операции.
*   `sort` (integer): Порядок сортировки.
*   `note` (text): Примечание.
*   `is_active` (boolean, NOT NULL): Статус активности (по умолчанию `true`).
*   `executor_id` (integer, FK): Ссылка на исполнителя (из табл. `executors`).

### Таблица `executors` (Исполнители)
Типы подразделений или ролей, выполняющих операции.
*   `id` (integer, PK): ID исполнителя.
*   `name` (text, NOT NULL): Название (например, Бизнес, Мидл-офис).
*   `note` (text): Описание.

### Таблица `systems` (Информационные системы)
Список ИТ-систем, используемых в операциях.
*   `system_id` (integer, PK): Автоинкрементный ID.
*   `system_name` (text, NOT NULL): Название системы (например, 1С, SAP).
*   `is_active` (boolean, NOT NULL): Статус активности (по умолчанию `true`).

## Таблицы приложения

### Таблица `users` (Пользователи)
*   `id` (integer, PK): Автоинкрементный ID.
*   `username` (text): Email или логин (уникальный).
*   `password_hash` (text): Хэш пароля (bcrypt).
*   `full_name` (text): Полное имя.
*   `role` (text): Роль (`admin` или `respondent`).
*   `is_active` (boolean): Активен ли пользователь.
*   `created_at` (timestamptz): Дата создания.

### Таблица `user_process_1_access` (Доступы)
Определяет, какие процессы L1 (и все их дочерние элементы) видит пользователь.
*   `user_id` (integer, FK): Ссылка на пользователя.
*   `f1_index` (text, FK): Ссылка на процесс L1.
*   Первичный ключ составной: `(user_id, f1_index)`.

### Таблица `user_answers` (Ответы пользователей)
Основная таблица с результатами замера.
*   `id` (bigint, PK): Автоинкрементный ID.
*   `user_id` (integer, FK): Ссылка на пользователя.
*   `operation_id` (text, FK): Ссылка на операцию (`process_4`).
*   `labor_hours` (numeric(10,2)): Трудозатраты в часах.
*   `system_id` (integer, FK): Используемая система.
*   `note` (text): Комментарий пользователя.
*   `is_done` (boolean): Отметка о завершении оценки.
*   `done_at` (timestamptz): Время завершения.
*   `created_at/updated_at`: Технические поля времени.

---

## Функции и триггеры

*   `copy_operations_to_user_answers(p_user_id)`: Автоматически создает пустые записи в `user_answers` для пользователя на основе его прав доступа в `user_process_1_access`.
*   `set_updated_at()`: Триггерная функция для автоматического обновления поля `updated_at`.
