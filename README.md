# ProcessMeter

Инструмент для проведения хронометража и оценки трудозатрат сотрудников в разрезе бизнес-процессов. Приложение позволяет пользователям оценивать время, затрачиваемое на конкретные операции, указывать используемые ИТ-системы и оставлять комментарии.

## Цели проекта

*   Сбор детальной статистики по трудозатратам (L4 уровень процессов).
*   Анализ использования ИТ-систем в рабочих процессах.
*   Выявление «узких мест» и потенциала для автоматизации.
*   Синхронизация данных с эталонными справочниками для обеспечения чистоты аналитики.

## Технологический стек

*   **Backend**: Node.js (v24), Fastify (v5), PostgreSQL.
*   **Frontend**: React 18, Vite, Vanilla CSS.
*   **Email**: Nodemailer (SMTP, настраивается через интерфейс администратора).
*   **DevOps**: Docker, Docker Compose.
*   **Связь**: REST API, JWT-авторизация (HttpOnly cookie).

## Основные возможности

1.  **Авторизация и безопасность**:
    * Разграничение прав на администраторов и респондентов.
    * JWT-токены в `HttpOnly` cookie (защита от XSS).
    * Rate limiting на публичных endpoint'ах (`/forgot-password`, `/token-info`).
    * Строгая валидация входящих запросов через Fastify JSON Schema.
    * Fail-fast: сервер не стартует в `production` без `JWT_SECRET`.

2.  **Приглашение пользователей и сброс пароля по email**:
    * Администратор отправляет приглашение или ссылку сброса пароля из интерфейса управления пользователями.
    * Одноразовые токены (24 ч) хранятся в таблице `password_tokens`; при повторной отправке старые инвалидируются транзакционно.
    * SMTP-настройки задаются в административном интерфейсе (раздел «Настройки») и хранятся в таблице `settings`.

3.  **Управление доступами**:
    * Администратор назначает, какие процессы оценивает конкретный пользователь.
    * Пакетная выдача и отзыв прав транзакционно безопасны.

4.  **Первичная инициализация**:
    * Синхронизация справочников (процессов, систем) из внутренней эталонной БД (`refdb`) через административный интерфейс.
    * Импорт выполняется напрямую между БД средствами SQL без использования сторонних ETL-сервисов.
    * Процесс импорта очищает и воссоздаёт `user_answers` под новую структуру.

5.  **Адаптивный интерфейс (UX)**:
    * Удобная работа с большими иерархическими списками процессов.
    * Skeleton Loaders для плавной загрузки.
    * Встроенная тёмная тема; сохранение UI-состояния в `localStorage`.

6.  **Производительность**:
    * Пакетное обновление через `INSERT ... FROM unnest` (без N+1).
    * Кеш транспортера Nodemailer с инвалидацией при изменении SMTP-настроек.
