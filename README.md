# ProcessMeter

Инструмент для проведения хронометража и оценки трудозатрат сотрудников в разрезе бизнес-процессов. Приложение позволяет пользователям оценивать время, затрачиваемое на конкретные операции, указывать используемые ИТ-системы и оставлять комментарии.

## Цели проекта

*   Сбор детальной статистики по трудозатратам (L4 уровень процессов).
*   Анализ использования ИТ-систем в рабочих процессах.
*   Выявление "узких мест" и потенциала для автоматизации.
*   Синхронизация данных с эталонными справочниками для обеспечения чистоты аналитики.

## Технологический стек

*   **Backend**: Node.js (v24), Fastify (v5), PostgreSQL.
*   **Frontend**: React, Vite, Vanilla CSS.
*   **DevOps**: Docker, Docker Compose.
*   **Связь**: REST API, JWT Authentication.

## Структура проекта

*   `/backend` - Серверная часть приложения.
    *   `/src` - Исходный код сервера и API.
    *   `/scripts` - Скрипты для миграции и обслуживания базы данных.
*   `/frontend` - Клиентская часть (SPA).
*   `/db` - Схема базы данных и документация по структуре.
*   `/old_tables` - Архивные данные для первичной инициализации.

## Основные возможности

1.  **Авторизация и Безопасность**:
    * Разграничение прав на администраторов и респондентов.
    * Защищенная авторизация через JWT с хранением токенов в `HttpOnly` cookie (защита от XSS-атак).
    * Строгая валидация входящих API-запросов с использованием Fastify JSON Schema.
2.  **Управление доступами**: Администратор назначает, какие процессы оценивает конкретный пользователь. Пакетная выдача прав транзакционно безопасна.
3.  **Первичная инициализация**: Синхронизация справочников (процессов, систем) из единой эталонной БД (`refdb`). Запускается при развертывании приложения, формируя безопасную изолированную структуру таблиц до старта работы респондентов. Импорт масштабируем (пакетная вставка с чанкованием обходит лимиты PostgreSQL).
4.  **Адаптивный интерфейс (UX)**:
    * Удобная работа с большими иерархическими списками процессов.
    * Сохранение пользовательского UI-состояния (состояние дерева процессов и ширина боковой панели) в `localStorage`.
    * Skeleton Loaders для плавной загрузки интерфейса без визуальных скачков.
    * Встроенная темная тема оформления.
5.  **Производительность**: 
    * Полное отсутствие проблем N+1 запросов за счет использования пакетного обновления `INSERT/UPDATE ... FROM unnest`.
    * Оптимизированный рендеринг: компоненты, такие как `ProcessTree`, обернуты в `React.memo` для предотвращения лишних перерисовок DOM при редактировании грида.
    * Проиндексированные внешние ключи БД для быстрого каскадного удаления и `JOIN` операций.
