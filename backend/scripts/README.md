# Описание скриптов бэкенда

В директории `backend/scripts` находятся служебные скрипты для управления базой данных, миграциями и пользователями.

## 1. `migrate.js`

**Описание:**
Основной скрипт для синхронизации данных между эталонной базой (`SOURCE_DATABASE_URL`) и рабочей базой приложения (`TARGET_DATABASE_URL`).
Выполняет следующие действия:
1. Сохраняет текущие права доступа пользователей во временную таблицу.
2. Очищает рабочие таблицы справочников.
3. Копирует актуальные справочники (`process_1`...`process_4`, `systems`, `executors`) из эталонной базы.
4. Восстанавливает права доступа пользователей.
5. Генерирует новые пустые формы ответов для всех активных пользователей.

**Запуск:**
```bash
node backend/scripts/migrate.js
# или через npm комманду из папки backend:
npm run db:migrate
```

## 2. `create-user.js`

**Описание:**
Утилита для создания нового пользователя через командную строку. Хэширует пароль и добавляет запись в таблицу `users`.

**Запуск:**
```bash
node backend/scripts/create-user.js <email> <password> "<Full Name>" [role]
```
*   `role` может быть `admin` или `respondent` (по умолчанию `respondent`).

**Пример:**
```bash
node backend/scripts/create-user.js test@example.com MySecretPass "Иван Иванов" admin
```

## 3. `clear-db.js`

**Описание:**
Скрипт для **полной очистки** базы данных. Удаляет абсолютно все данные: пользователей, настройки доступов, ответы, справочники процессов и системы.
Используется для полного сброса состояния приложения.

**ВНИМАНИЕ:** Действие необратимо!

**Запуск:**
```bash
node backend/scripts/clear-db.js
# Скрипт запросит подтверждение (введите "yes")
```
